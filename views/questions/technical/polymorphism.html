What is polymorphism, and why is it important in OOP?

Polymorphism comes from the Greek roots "poly," which means "many" and "morphe," which means "forms." It's a concept that allows for the construction of general data structures and methods, which is invaluable in OOP.

In the context of programming, polymorphism is the capacity of a given object to be regarded as an instance of its own class, an instance of its superclass, or an instance of its superclass's superclass (and so on and so forth up the hierarchy). This is valuable because it allows for the creation of interfaces that can be used to provide an instruction set of sorts for numerous subclasses. In other words, it makes it possible to use the same general formulae to construct similar objects, which means we don't need to rewrite the same functionality over and over again. This minimizes the potential for some miniscule difference being introduced (as will invariably occur when different people work on the same code) into the codebase. It may not sound like a huge problem, but it's actually one of the most insidious, ubiquitous, and apoplectic-rage inducing problems in team programming. It's the reuse and recycling of code that polymorphism enables that makes it so valuable in programming!
