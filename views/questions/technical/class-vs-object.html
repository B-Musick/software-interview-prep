Classes and Objects in OOP

More technical interview questions

What are classes and objects in OOP, and what's the distinction between the two?

A Class is a blueprint of sorts; it outlines what attributes and functionalities a particular type of structure should have. For example, a Car object could have color, top speed, and transmission type attributes and accelerate, brake, and turn signal functions.

An Object is an instance of a Class. For instance (if you'll excuse the pun), myCar would be an instance of the class Car with color = gray, top speed = 15mph, transmission type = automatic. It would inherit the methods accelerate, brake, and turn signal functions from Car.

An object is a member or an "instance" of a class.

If channel is a class, Start Sports, BBC, and ESPN are its objects. If water is a Class, "Fresh Lime Water" and "Soup" are its objects.
